    又到了使用zookeeper的时候了，虽然我对他不熟。
    我很讨厌zookeeper，不单单是CP的原因。
    平时测试zookeeper不想用云端的，开win版本的，结果会再本磁盘创建缓存文件，还比较大，欺负我不懂它的配置。
    隔了一段时间了，打开zookeper的zip包，打开报错

    1. 没有JDK环境，于是我去zkEnv.md里面补了一句SET JAVA_HOME="D:\JAVA\JDK\jdk1.8.0"

    2. 错误: 找不到或无法加载主类 org.apache.zookeeper.server.quorum.QuorumPeerMain,好家伙，一搜索，这不是可以使用的版本，于是下bin的zip包。

    3. 下好windows包，打开依然报错，嗯，知道了，配置文件的文件名不对，但是我会改配置文件的文件名嘛？明显不会

      @REM setup java environment variables

@REM 配置JDK
SET JAVA_HOME="D:\JAVA\JDK\jdk1.8.0"
@REM 释放配置文件
cd ../conf
if exist "zoo_sample.cfg" (
ren "zoo_sample.cfg" "zoo.cfg"
)
cd ../bin

      是的，你没看错，在/bin目录下的zkEnv.md文件总@REM setup java environment variables的下一行增加了如上的内容，反正不会手动重命名。

      配置文件保存一下，再也不想配置Zookeeper了。
