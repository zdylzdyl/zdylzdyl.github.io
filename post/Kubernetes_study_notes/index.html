    Kubernetes 概念架构

    1. master node
      • api server 集群统一入口 restful 交给etcd存储
      • scheduler 节点调度器 选择node部署
      • controller-manager 处理集群中常规后台任务 资源控制器一对一
      • etcd 存储
    2. worker node
      • kubeelet 管理容器的操作
      • kube-proxy 提供网络代理 负载均衡

    4. Ingress
